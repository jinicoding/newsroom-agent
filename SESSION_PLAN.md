### 전략적 판단

11:00 세션에서 공공정보 접근(press, law)과 품질 측정(readability)을 완성했다. 파이프라인은 취재→리서치→분석→작성→편집→출고→아카이브→협업→현황판까지 59개 커맨드로 기자 워크플로우 전 과정을 커버한다. 하지만 세 가지가 빠져 있다:

1. **편집 데스크 수준의 기사 개선 피드백** — /proofread는 맞춤법, /readability는 점수를 매기지만, "이 리드를 이렇게 바꾸면 더 낫다"는 구체적 개선안이 없다.
2. **취재 일정 관리** — /deadline은 마감 타이머 하나뿐. 기자간담회, 국회 일정, 재판 기일 등 복수 일정을 관리할 달력이 없다.
3. **실시간 여론 감지** — /news와 /alert는 언론 보도 중심. "사람들이 지금 뭘 얘기하는가"(SNS 트렌드)를 포착하는 도구가 없다.

이 세 가지는 "기사의 질을 높이고, 취재를 놓치지 않고, 독자를 이해하는" 영역이다. 기자가 매일 수작업으로 하는 일들이며, 자동화의 가치가 높다.

### Task 1: /improve — AI 기반 기사 개선 제안
Files: `src/commands_project.rs`, `src/commands.rs`
Description: 기사 텍스트(파일 경로 또는 최근 초안)를 AI에게 넘겨 편집 데스크 수준의 구체적 개선안을 제시하는 커맨드. /proofread가 맞춤법·문체 교정이고 /readability가 가독성 점수 측정이라면, /improve는 "이 기사를 어떻게 하면 더 좋은 기사로 만들 수 있나"에 답하는 편집 피드백 도구다. 분석 항목: (1) 리드문 흡인력, (2) 문단 전환 자연스러움, (3) 인용문 활용도, (4) 구체성·수치 뒷받침, (5) 결론/마무리 완성도. 각 항목에 대해 현재 상태 평가 + 구체적 수정 제안을 제공한다. 결과는 `.journalist/improve/`에 저장해 이력 추적 가능. 테스트: 인자 파싱, 파일 읽기, 프롬프트 구성 검증.
Issue: none

### Task 2: /calendar — 취재 일정 관리
Files: `src/commands_project.rs`, `src/commands.rs`
Description: 기자간담회, 국회 일정, 재판 기일, 발표 일정 등 취재 관련 일정을 등록·조회·관리하는 커맨드. AI 호출 없이 로컬에서 동작. 하위 명령: `add <날짜> <시각> <설명>` (일정 등록), `list` (전체 목록, 날짜순 정렬), `today` (오늘 일정), `week` (이번 주 일정), `done <번호>` (완료 처리), `remove <번호>` (삭제). /deadline이 마감 시각 하나를 추적하는 타이머라면, /calendar는 다수의 취재 일정을 체계적으로 관리하는 달력이다. `.journalist/calendar.json`에 JSON 배열로 저장. 오늘/내일 임박 일정은 색상 코딩으로 강조(빨강: 오늘, 노랑: 내일). 테스트: CRUD 연산, 날짜 파싱, 정렬 로직 검증.
Issue: none

### Task 3: /sns — SNS 트렌드 모니터링
Files: `src/commands_project.rs`, `src/commands.rs`
Description: 소셜미디어 실시간 트렌드와 키워드별 여론을 탐지하는 커맨드. 하위 명령: `trend` (실시간 트렌드 키워드 — curl로 공개 트렌드 페이지 파싱), `search <키워드>` (키워드의 SNS 반응·여론을 AI가 분석), `buzz <키워드>` (화제성 분석 — 최근 뉴스와 SNS 반응을 종합해 AI가 화제성 판단). /news가 언론 보도 검색이고 /alert가 뉴스 모니터링이라면, /sns는 "사람들이 지금 뭘 얘기하고 있나"를 포착하는 여론 감지기다. 속보 판단, 기사 앵글 발굴, 독자 반응 파악에 활용. 결과는 `.journalist/sns/`에 캐시. 테스트: 인자 파싱, 하위 명령 라우팅, 캐시 경로 검증.
Issue: none
